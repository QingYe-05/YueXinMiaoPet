# 安全策略

## 支持版本

当前主要维护版本：

| 版本 | 支持状态 |
| --- | --- |
| 1.0.x | 支持 |

## 报告安全问题

如果你发现安全相关问题，请不要直接公开具体利用细节。可以通过以下方式联系维护者：

- 在 GitHub 仓库中创建一个 Issue，标题中写明 `[Security]`，并尽量避免包含敏感细节。
- 如果问题涉及用户本地文件、安装脚本、配置文件写入、自动启动、下载或更新逻辑，请在报告中说明影响范围和复现条件。

## 安全关注点

本项目重点关注：

- 用户配置只写入 AppData，不写入安装目录。
- 不主动上传用户本地数据。
- 天气接口失败时安全回退，不导致程序崩溃。
- GIF、配置和 JSON 读取失败时安全回退。
- 安装脚本避免误删用户文件。
- 自定义 GIF 目录只读取本地 GIF，不执行其中内容。

## 处理流程

维护者会尽量按以下流程处理：

1. 确认问题是否可复现。
2. 判断影响范围。
3. 修复并测试。
4. 在 CHANGELOG 中记录修复说明。
5. 如有必要，发布修复版本。

## 不属于安全问题的情况

以下问题通常按普通 Bug 或功能建议处理：

- GIF 分类不准确。
- 天气展示不明显。
- UI 布局问题。
- 某些 Windows 版本上的显示兼容性问题。
